# Teal 暗色主题与三段切换设计

## 背景
当前前端只有默认主题与 Teal 亮色主题两种状态，主题切换按钮也只在两者之间切换。本需求要在保留 Teal 作为基础色调的前提下，新增一个完整的 Teal 暗色模式，并把主题切换改成三段点击循环。

## 目标
- 提供 `默认 -> Teal 亮色 -> Teal 暗色 -> 默认` 的循环切换。
- Teal 暗色覆盖全站用户可见区域：页面、弹窗、抽屉、按钮、下拉、高亮态、选中态。
- 暗色模式下的浅色文字略微加粗，减少发虚感。
- 保持现有主题存储与启动链，不引入新的全局状态源。

## 非目标
- 不改路由结构，不新增页面。
- 不重做 UI 布局，只做主题与状态样式覆盖。
- 不引入第二套主题配置来源。

## 主题状态
保留现有 Teal 亮色作为中间态，新增 Teal 暗色作为第三态。建议继续用 `localStorage` 保存主题值，并兼容历史值。

```mermaid
stateDiagram-v2
  default --> teal: click
  teal --> teal-dark: click
  teal-dark --> default: click
```

### 兼容规则
- 旧的 `teal` 存储值继续表示 Teal 亮色。
- 非法值统一回落到 `default`。
- 进入应用时先同步主题，再挂载应用，避免闪烁。

## 视觉系统
### 基础原则
- 默认主题保持现有浅色观感。
- Teal 亮色继续沿用现有 Teal 方向，但改为统一 token 化。
- Teal 暗色使用深青黑底色，Teal 作为唯一强强调色。
- 深色模式下，文字、按钮、标题、菜单项、选中态的浅色字重整体略微上提。

### 建议 token
- `background`
- `surface`
- `surface-2`
- `border`
- `text`
- `text-muted`
- `accent`
- `accent-strong`
- `hover`
- `selected`
- `focus-ring`

## 覆盖范围
### 页面
- 登录页
- 聊天页
- 管理后台各页
- 个人设置弹窗
- 开源许可弹窗

### 组件与状态
- 弹窗、抽屉、确认层、菜单层
- 主按钮、幽灵按钮、图标按钮、禁用态
- `select`、下拉菜单、弹出层、选项高亮
- 表格行、列表项、侧栏选中态
- 输入框、焦点环、滚动条

## 主题按钮
- 保留侧栏中的主题切换入口，仍然是轻量图标按钮。
- 点击一次推进到下一个主题状态，不再是二段切换。
- 按钮的可访问名称和 tooltip 需要明确提示“下一次点击会切换到哪里”。
- 按钮视觉上建议使用三段指示器，当前段高亮，帮助用户理解循环状态。

## 实施边界
### 共享层
- `webapp/src/lib/theme.ts` 负责主题状态、存储、应用与循环逻辑。
- `webapp/src/style.css` 负责全局 token、默认主题、Teal 亮色、Teal 暗色和常见控件覆盖。

### 组件层
- 侧栏主题按钮根据当前状态更新图标与文案。
- 其余页面尽量只吃全局 token，不单独复制主题逻辑。
- 对 Teleport 到 `body` 的弹窗/下拉，必要时补充 `.theme-teal-dark` 作用域覆盖。

## 测试与验收
- 主题工具单测覆盖三段循环、旧值兼容、非法值回落。
- 侧栏主题按钮交互测试覆盖三次点击的状态推进。
- 手工或浏览器验证以下场景：
  - 登录页
  - 聊天页
  - 管理页
  - 个人设置弹窗
  - 开源许可弹窗
  - 下拉高亮与选中态
  - 抽屉/侧栏

## 风险
- 部分组件 scoped CSS 里可能残留硬编码浅色值，需要逐个收口到主题 token。
- Teleport 到 `body` 的浮层如果只靠局部类选择器，可能漏样式，需要补全全局作用域。
- 深色模式下字体权重提升要适度，避免标题和按钮看起来过重。
